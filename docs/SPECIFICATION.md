## 6. Информация о программе

### 6.1. Общее описание

Программа **«Матье»** — клиент к базе данных `matye`, обеспечивающий:

- **Гостю** — просмотр каталога услуг с фильтрами и пагинацией;
- **Пользователю** — профиль, баланс (имитация пополнения с карты), запись на услугу к мастеру, номер очереди, отзывы;
- **Модератору** — управление привязкой мастеров к услугам, рассмотрение заявок на повышение квалификации, создание и редактирование услуг (с обновлением `modified_at`);
- **Мастеру** — просмотр своих услуг и записей клиентов, подача заявки на повышение квалификации;
- **Администратору** — управление пользователями и добавление сотрудников.

### 6.2. Состав решения

| Компонент | Описание |
|-----------|----------|
| `KvalikSamira` | Основной проект приложения (UI, ViewModels, доступ к данным) |
| `KvalikSamira.Tests` | Модульные тесты (xUnit), в т.ч. логика времени последнего изменения услуги |
| `TESTCASES.md` | Таблицы тест-кейсов (ручные и привязка к автотестам) |

### 6.3. Зависимости (основные)

- Avalonia 11.3.12, Fluent theme, шрифты Inter  
- Entity Framework Core 8.0.11, Npgsql EF Provider 8.0.11  
- CommunityToolkit.Mvvm 8.2.1  

### 6.4. Запуск

1. Развернуть БД PostgreSQL по скрипту `database.sql` (или восстановить из бэкапа).  
2. При необходимости изменить строку подключения в `MatyeDbContext.cs`.  
3. Собрать решение: `dotnet build KvalikSamira.sln`.  
4. Запустить `KvalikSamira.exe` из каталога сборки.

| Объект | Инструмент | Результат |

### 7.2. Тест-кейсы и ручное тестирование

Документ **`TESTCASES.md`** (в корне папки решения `KvalikSamira`) содержит:

- детальные **unit**-кейсы для модуля времени изменения (секция A);  
- по **5 сценариев** на основные функциональные блоки приложения (авторизация, регистрация, каталог, профиль, модератор, мастер, администратор и др.) с шагами, ожидаемым и фактическим результатом (фактический результат для ручных кейсов заполняется при прогоне в конкретной среде).

### 7.3. Вывод

Реализованный модуль определения и записи времени последнего изменения услуги **покрыт автоматическими тестами** и используется в коде сохранения услуги (`ServiceListViewModel`). Остальной функционал проверяется по чек-листу в **`TESTCASES.md`** методом ручного или полуавтоматического тестирования на стенде с рабочей БД.
